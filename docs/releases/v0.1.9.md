## Requirements

- Ubuntu 24.04 **amd64**
- Docker + Compose plugin (ставятся как зависимости пакета)
- Python 3.12+, Tkinter, 7z (`p7zip-full`) — тоже в зависимостях `.deb`
- Telegram: приложение на [my.telegram.org](https://my.telegram.org) + приватная группа для бэкапов

## Clean install

### 1. Установка пакета

Скачайте `telegram-uploader_0.1.9_amd64.deb` из Assets этого релиза.

```bash
sudo apt update
sudo apt install ./telegram-uploader_0.1.9_amd64.deb
telegram-uploader-check-deps
```

Если `check-deps` сообщает, что Docker недоступен:

```bash
sudo systemctl enable --now docker
sudo usermod -aG docker "$USER"
```

Выйдите из сессии и войдите снова (или перезагрузите ПК), затем:

```bash
telegram-uploader-check-deps
```

Если ставили через `dpkg -i` и зависимости не подтянулись:

```bash
sudo apt -f install
sudo apt install docker.io docker-compose-plugin python3-venv python3-tk p7zip-full
telegram-uploader-check-deps
```

### 2. Конфиг приложения (один раз)

```bash
telegram-uploader --setup
```

Создаёт `~/.config/telegram-uploader/.env`.

### 3. Подготовка Telegram

| Что нужно | Где взять |
|-----------|-----------|
| API ID + API hash | [my.telegram.org](https://my.telegram.org) → API development tools |
| Группа для бэкапов | Приватная группа в Telegram; ваш аккаунт должен быть участником |
| Target chat ID | Id **группы**, обычно `-100…` (не ваш личный user id) |

Подробнее: [TELEGRAM_SETUP.md](https://github.com/RomFuture/telegram-uploader/blob/main/docs/TELEGRAM_SETUP.md).

### 4. Запуск

```bash
telegram-uploader
```

Команда поднимает Postgres, Redis и Celery-воркеры в Docker, применяет миграции БД и открывает GUI.

### 5. Первый запуск в GUI

1. **Unlock** — ключ шифрования, если задавали в Settings.
2. **Settings → Client API** — API ID, API hash; путь сессии по умолчанию: `~/.config/telegram-uploader/session.session`.
3. **Settings → General** — **Target chat ID** = id группы (`-100…`).
4. **Save**.
5. **Sign in to Telegram…** в Settings **или** в терминале:

   ```bash
   telegram-uploader-login
   ```

   (телефон + код из Telegram, один раз)

6. **Test Client API** — в Settings; тестовый файл должен появиться в группе.
7. **Add file → Backup** — первый бэкап.

| Файл | Путь |
|------|------|
| Конфиг | `~/.config/telegram-uploader/.env` |
| Сессия Telegram | `~/.config/telegram-uploader/session.session` |
| Файлы приложения | `/opt/telegram-uploader/` |

Полная инструкция: [README — Clean install](https://github.com/RomFuture/telegram-uploader#clean-install-first-time).

## Upgrade

Обновление с предыдущей версии `.deb` (настройки и сессия сохраняются):

```bash
# Закройте GUI, если открыт
docker compose -f /opt/telegram-uploader/docker-compose.yml down
sudo apt install ./telegram-uploader_0.1.9_amd64.deb
telegram-uploader
```

Повторный `--setup` и вход в Telegram **не нужны**, если не меняли аккаунт и не удаляли session-файл.

См. [README — Upgrading](https://github.com/RomFuture/telegram-uploader#upgrading-deb-users).
