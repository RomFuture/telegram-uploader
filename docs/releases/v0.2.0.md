## Requirements

- Ubuntu 24.04 **amd64**
- Docker + Compose plugin (ставятся как зависимости пакета)
- Python 3.12+, Tkinter, 7z (`p7zip-full`) — тоже в зависимостях `.deb`
- Telegram: приложение на [my.telegram.org](https://my.telegram.org) + приватная группа для бэкапов

---

## Clean install

### 1. Установка пакета

Скачайте `telegram-uploader_0.2.0_amd64.deb` из Assets этого релиза.

```bash
sudo apt update
sudo apt install ./telegram-uploader_0.2.0_amd64.deb
telegram-uploader-check-deps
```

Если `check-deps` сообщает, что Docker недоступен:

```bash
sudo systemctl enable --now docker
sudo usermod -aG docker "$USER"
```

Выйдите из сессии и войдите снова (или перезагрузите ПК), затем:

```bash
telegram-uploader-check-deps
```

### 2. Конфиг приложения (один раз)

```bash
telegram-uploader --setup
```

### 3. Запуск

```bash
telegram-uploader
```

Полная инструкция: [README — Clean install](https://github.com/RomFuture/telegram-uploader#clean-install-first-time) · [TELEGRAM_SETUP.md](https://github.com/RomFuture/telegram-uploader/blob/main/docs/TELEGRAM_SETUP.md).

## Upgrade

```bash
docker compose -f /opt/telegram-uploader/docker-compose.yml down
sudo apt install ./telegram-uploader_0.2.0_amd64.deb
telegram-uploader
```

Повторный `--setup` и вход в Telegram **не нужны**, если не меняли аккаунт.

## Что нового в 0.2.0

### Архитектура и use cases

- Переименования public API: `BackupApi` → `GuiEntrypoint`, `WorkerApi` → `CeleryEntrypoint`; `wire_gui_entrypoint` / `wire_celery_entrypoint`.
- `test_client_api` → `verify_storage_provider` — сценарий проверки Client API без привязки к конкретному провайдеру.
- **`session/` refactor:** `create.py` (CreateSession, CreateDatabase, CreateFolder), `list.py` (ListProfiles, ListFolders); меньше мелких файлов.
- **`GetSessionProgress` → `GetSessionQueueSnapshot`** — имя отражает снимок очереди, а не progress bar; цепочка DTO/API переименована (`SessionQueueSnapshotDTO`, `get_queue_snapshot`, `render_queue_snapshot`).
- **`DomainError` factories:** `required()`, `folder_already_exists()`, `folder_not_found()`; единый стиль ошибок в use cases.
- **`domain/MANUAL.md`** + docstring'и на actions, errors, models — hover-подсказки в IDE.
- **`use_cases/MANUAL.md`**, **`DEBUG_BACKUP.md`** — путеводитель и breakpoint-лист для backup pipeline.
- **`manage_source_item`:** корректные ошибки (`required`, `folder_not_found`) вместо misuse `invalid_status_transition`.

### Restore

- Restore preflight (`check_restore_ready`), scope по папкам, progress download, валидация dest path.
- GUI restore с выбором папки назначения; worker hook `ProcessRestoreVolumeUseCase`.

### Infrastructure

- Structured logging (`observation/logging_setup.py`), `APP_LOG_LEVEL`, worker logging on Celery `worker_process_init`.
- Restore/backup fixes: Client API refs, legacy volume detection, seven-zip edge cases.

### Тесты и CI

- Расширенное покрытие restore, scope, verify_storage_provider, logging.
- Integration test: уникальное имя профиля (не падает при повторном прогоне на Postgres с UNIQUE).
- Mypy: fix untyped Celery signal decorator.

### Документация

- Обновлены `PROJECT.md`, `README.md`, `BACKLOG.md`; удалены устаревшие гайды.
- Release notes для `.deb` пользователей (этот файл).
