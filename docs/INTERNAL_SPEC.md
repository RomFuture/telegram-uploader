# Internal Specification (Non-Negotiable Rules)

This document defines mandatory product constraints for implementation.

## 1. Interface Language Policy

- All user-facing text must be in English.
- Applies to GUI labels, buttons, tooltips, dialogs, validation errors, user notifications,
  and logs visible in UI.

## 2. Telegram Usage Scope (`v1`)

- Upload destination is one target Telegram group.
- Topic semantics are forbidden (`message_thread_id` must not be used).
- **Provider (migration in progress):** production target is **Telegram Client API** (user MTProto session) for reliable restore from the group. Bot API adapter remains legacy until [TELEGRAM_CLIENT_API_MIGRATION.md](TELEGRAM_CLIENT_API_MIGRATION.md) completes.

## 3. Security Policy

- Archive encryption is mandatory.
- Encryption key is auto-generated by default.
- User can override/change encryption key in settings.

## 4. Restore Metadata Persistence

After upload, the system must persist:

- `external_file_id`
- `external_message_id`
- `provider_download_ref`

These identifiers are required for restore flows.

## 5. Filename Policy

- Outgoing Telegram archive volume names must include a hashed user filename component.
- The hash must be computed from `display_name` captured at enqueue time, not from the path on disk at archive time.

## 6. Display Name Policy

- The original user-visible filename (e.g. `Мысли.txt`) must be captured as `display_name` in `source_items` at the moment the item is enqueued.
- `display_name` is immutable after enqueue.
- GUI must always show `display_name`; it must never derive the visible name from `source_path` at render time.

## 7. Source Files Policy

- The app must not auto-move source files into a service directory.
- User controls data organization through profile/workspace in GUI.
